# Example Profile

This is a synthetic placeholder profile for Research Lab. It demonstrates the profile shape without exposing a real company, client, source corpus, strategy, or private decision context.

## Project

Example Project is a fictional product team evaluating a new customer-facing research workflow. Replace this file before running real research.

## Decision Owner And Audience

- Decision owner: project owner
- Default audience: product, strategy, or operations leadership
- Durable context destination: project knowledge base or ChatGPT project context

## Decision Taxonomy

Use this taxonomy for the example profile unless a run brief narrows it:

1. Market decision
2. Product decision
3. Customer research decision
4. Competitive positioning decision
5. Pricing or packaging decision
6. Vendor selection decision
7. Technical architecture decision
8. Legal or policy decision
9. Operations decision

If a research idea does not change one of these decisions, it waits.

## Internal Context Rule

Profile docs are context and hypotheses, not external proof. External factual claims still require external citations from public sources, official APIs, app listings, reports, reviews, policies, papers, or other cited evidence.

Required source context must be available before synthesis. If source context is missing, stop and report exactly what is missing, where Codex looked, and what is needed next. Do not hallucinate missing source context.
