# Security

Do not report secrets, credentials, private source documents, account exports, generated research outputs, or sensitive exploit details in a public issue.

## Reporting

Before making this repository public, enable GitHub private vulnerability reporting or add a project-specific private reporting contact here.

Until that is configured:

- Open public issues only for non-sensitive documentation, packaging, or workflow problems.
- Do not paste tokens, API keys, private URLs, raw datasets, or account identifiers.
- If you discover committed sensitive material, stop and notify the project owner through a private channel.

## Research Boundary

Default research runs must not mutate production accounts or external systems. Do not use billing tools, production databases, deployment systems, posting/sending tools, private account actions, or access-control bypasses unless a run brief explicitly scopes an approved, documented process.
