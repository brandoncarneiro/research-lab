# Research Backlog

This root backlog is intentionally generic. Each project profile owns its prioritized backlog so the runtime does not force every run to be about one project.

## Default Backlog

The included example backlog is:

```text
profiles/example/RESEARCH_BACKLOG.md
```

Use it as a shape reference only. Real projects should copy `profiles/example/` and maintain their own profile backlog.

## Backlog Rules

- Choose one decision-grade run at a time unless the run brief explicitly scopes a combined run.
- Every backlog item must name the decision affected.
- The active profile may define its own decision taxonomy, default lanes, source context, and first-run priority.
- Core evidence rules still apply across all profiles: cited factual claims, confidence labels, contradictions, negative evidence, blocked-source logging, and no synthesis before raw lane evidence exists.
